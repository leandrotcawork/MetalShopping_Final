# Platform Package Checklist

## Before editing

- confirm the capability belongs in `platform/`
- confirm package purpose is narrow and explicit

## While editing

- keep business semantics out of platform code
- keep operational concerns visible
- keep interfaces explicit where reuse matters

## Final review

- package fits `docs/PLATFORM_PACKAGE_STANDARDS.md`
- boundary decisions fit `docs/PLATFORM_BOUNDARIES.md`
- operational behavior fits `docs/OBSERVABILITY_AND_SECURITY_BASELINE.md`
