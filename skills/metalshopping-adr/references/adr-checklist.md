# ADR Checklist

## Stage 1 — Write
- [ ] File: `docs/adrs/ADR-XXXX-<title>.md`
- [ ] Status: Proposed
- [ ] Context explains why decision was needed
- [ ] Decision is precise (not vague)
- [ ] Consequences covers trade-offs
- [ ] Acceptance test is concrete and runnable (not "looks correct")

## Stage 2 — SOT
- [ ] `docs/PROJECT_SOT.md` updated
- [ ] No contradictions with existing SOT entries

## Stage 3 — Verify
- [ ] Acceptance test executed and passes
- [ ] Implementation fixed if test failed first

## Stage 4 — Commit
- [ ] ADR Status updated to Accepted
- [ ] Commit: `docs(adr): ADR-XXXX <title> — verified and closed`
- [ ] Both ADR file and PROJECT_SOT in the commit
