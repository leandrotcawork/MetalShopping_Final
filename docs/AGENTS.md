# AGENTS

## Scope

This file applies to `docs/`.

## Documentation rules

- keep docs short, explicit, and non-duplicative
- prefer ADRs for binding decisions
- prefer `PROJECT_SOT.md` for the current operational truth
- prefer `PROGRESS.md` for phase status and next steps

## Update discipline

- if a platform rule changes, update the ADR first
- if phase or execution order changes, update `IMPLEMENTATION_PLAN.md` and `PROGRESS.md`
- avoid creating new planning files when an existing SoT doc should be updated instead
