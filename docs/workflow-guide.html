Skills & Plugins

Custom skills (agent routing) and installed plugins

Custom Skills
$msMaster Orchestrator
Entry point for every task. 4 decisions before any T-stage. Never calls analytics agents directly.
InvokeEvery task by default
InputsTask description from user
OutputsDecision log in todo.md + executed T-stage chain
$metalshopping-docsT7 Docs Agent
Delta-only documentation update. Updates only affected sections. Non-blocking subagent dispatched at T7 in parallel with Review.
InvokeDispatched by $ms at T7 (non-blocking)
OutputsUpdated doc sections only
$analytics-orchestratorAnalytics Entry
Analytics domain entry point. Routes all analytics tasks to the correct sub-agent.
InvokeRouted from $ms for any analytics scope
OutputsRoutes to sub-agents, returns summary to $ms
$analytics-intelligenceSTUB
STUB: 8-layer intelligence model. Scoring, ranking, and insight generation.
InvokeDispatched by $analytics-orchestrator
$analytics-surfacesSTUB
STUB: 11 analytics read surfaces — frontend views and their backend query patterns.
InvokeDispatched by $analytics-orchestrator
$analytics-campaignsSTUB
STUB: Campaigns and alerts analytics layer.
InvokeDispatched by $analytics-orchestrator
$analytics-aiSTUB
STUB: AI copilot layer — recommendations, anomaly explanations, natural-language queries.
InvokeDispatched by $analytics-orchestrator
$workflow-guardianInspect & Report
Inspect-and-report workflow health. Checks T-stage compliance, lesson accumulation, ADR alignment. Never auto-updates — reports only.
InvokeADR committed / 3+ lessons in domain / every 10 tasks / /workflow-check
OutputsHealth report (no file changes)
Installed Plugins
commit-commands
/commit slash command. Structured commits in <type>(<scope>): <what> format.
Step 4 of post-task sequence. Also callable anytime via /commit.
claude-mem
Cross-session context compression. Produces session artifacts recoverable at next session start.
Step 6 of post-task sequence. Session start reads the artifact when context was compacted.
claudewatch
Real-time monitoring: drift detection, cost velocity, MCP tools (get_drift_signal, get_session_dashboard, get_project_health, get_blockers, extract_current_session_memory).
Always active. Session start calls get_project_health. PostToolUse hook evaluates writes.
code-review / pr-review-toolkit
/code-review slash command. 5 specialized review agents: code-reviewer, code-simplifier, comment-analyzer, pr-test-analyzer, silent-failure-hunter.
Run /review-pr before merging significant branches.
security-guidance
Injects security rules into every write. Guards against OWASP top 10, command injection, SQL injection, XSS.
Always active via PreToolUse hook on file writes.
claude-md-management
Updates CLAUDE.md post-task with learnings from the session. Keeps project instructions accurate.
Step 5 of post-task sequence via /revise-claude-md.
superpowers
Planning, subagent execution, brainstorming, debugging, and verification. Skills: writing-plans, executing-plans, subagent-driven-development, brainstorming, systematic-debugging.
Plan mode → writing-plans. Complex research → dispatching-parallel-agents. Debugging → systematic-debugging.
skill-creator
Creates and improves skills. Authors new SKILL.md files, runs evals, optimizes skill descriptions for triggering accuracy.
Run when adding or refining agent skills in .agents/skills/.
frontend-design
Production-grade frontend UI generation. Avoids generic AI aesthetics. New pages and visually complex components.
Used at T5 for architectural or visually complex frontend work.
feature-dev
Guided feature development with architecture focus. Explores codebase patterns before proposing implementation.
Used at T2/T5 when exploring unfamiliar codebase areas before writing code.
context7
Up-to-date library documentation via resolve-library-id + query-docs. Live docs for any npm package or Go module.
Used during any T-stage when library documentation is needed.
figma
Figma design → production code with 1:1 visual fidelity. Also: code-connect-components, create-design-system-rules.
Used at T5 when Figma designs are the source of truth for frontend implementation.
pr-review-toolkit
Comprehensive PR review: code-reviewer, code-simplifier, comment-analyzer, pr-test-analyzer, silent-failure-hunter, type-design-analyzer.
Run /review-pr before merging any significant branch to main.

New backend module from scratch
$ms→Plan YES→T1 Opus→T2 Sonnet→T3a Codex→T3b Claude→T3.5→T4 Script→T5 Split→T5.5 Codex→T6+T7+Review→Commit
"Add procurement order tracking module." Plan YES (new module + tenant isolation). Opus for T1 (new domain), Sonnet for rest. Codex on T3a + T5.5. T6 ADR written (new domain pattern).
# Decision log (tasks/todo.md):
Plan: YES — new module, >2 files, touches tenant isolation
Model: Opus (T1 new domain) → Sonnet (remainder)
Codex: YES for T3a adapters + T5.5 tests
Parallel: YES — T6 ADR + T7 + Review at end

# T4 — run script (not AI):
./scripts/generate_contract_artifacts.ps1 -Target all
2
Frontend bug fix
$ms→Plan NO→T5 Claude→T5.5 Codex→Review Gate→Commit
"Spotlight table headers lose hover state after sort." Plan NO (single CSS fix, Lesson 20 pattern). Sonnet. T5 Claude only. T5.5 Codex adds regression test.
# Decision log:
Plan: NO — single CSS fix, existing pattern (Lesson 20)
Model: Sonnet
Codex: YES for T5.5 test only
Parallel: NO — sequential (T5 → T5.5 → Review)
3
Analytics task routing
$ms→Analytics scope detected→$analytics-orchestrator→$analytics-surfaces→Review→Commit
"Add margin band filter to Produtos analytics table." $ms checks: touches packages/feature-analytics → route to $analytics-orchestrator → identifies surface work → $analytics-surfaces.
# $ms routing check:
Task touches: packages/feature-analytics/src/products/
→ Analytics routing: YES
→ Route to: $analytics-orchestrator
→ NEVER call $analytics-surfaces directly from $ms
4
Codex handoff (T3a adapters)
T3a → Codex decision→Write codex-handoff.md→Codex implements→Claude reviews→Fix if needed→Commit + delete handoff
After T2 defines ports, T3a qualifies for Codex (plan complete, non-critical files). Claude writes the handoff spec. Codex loads it with lessons.md.
# tasks/codex-handoff.md:
Task: Implement procurement order PostgreSQL adapter
Files:
  Create: apps/server_core/internal/modules/procurement/adapters/order.go
  Create: apps/server_core/internal/modules/procurement/transport/order.go

Constraints:
  - Use pgdb.BeginTenantTx on every query (Lesson 1)
  - Use current_tenant_id() in every WHERE clause
  - Call PrincipalFromContext first in every handler
  - Outbox: use outbox.AppendInTx in same tx as INSERT (Lesson 3)

Pattern: apps/server_core/internal/modules/pricing/adapters/

Definition of done:
  go test ./apps/server_core/internal/modules/procurement/... passes

Session start output example
Hook fires→Read 3 files→Output STATE/NEXT/WATCH→Wait
Real output when Track B is in-progress and two relevant lessons are active.

  Track B — Spotlight parity (visual audit pending)
  Last commit: fix(analytics): restore spotlight sort interactions

NEXT:
  Run diff against legacy_snapshot for remaining CSS deltas
  File: packages/feature-products/src/spotlight/SpotlightTable.module.css

WATCH:

  Lesson 5  — Frontend data flow must use platform SDK contracts
6
Lesson quality filter — three examples
Task completes→Filter runs (3 gates)→Write or skip
All three gates must be YES. Two failing gates means skip — no partial lessons.
EXAMPLE A — PASSES (write to lessons.md):
"Go adapter query without BeginTenantTx causes cross-tenant data reads"
✓ Would cause a bug if repeated (data leak)
✓ Code logic / architectural (Go adapter layer)
✓ Recurring pattern (every adapter must follow this)
→ WRITE

EXAMPLE B — FAILS (skip):
"Spotlight header background was missing border-radius"
✗ Not a bug — cosmetic UI adjustment
→ SKIP

EXAMPLE C — FAILS (skip):
"Removed unused import in one migration file"
✗ One-off cleanup, not a recurring pattern
→ SKIP

Post-task Sequence
1.Mark [x] in tasks/todo.md
2.Lesson quality filter evaluation
3.Memory evaluation → update MEMORY.md if applicable
4.Commit via commit-commands: <type>(<scope>): <what>
5.claude-md-management updates CLAUDE.md
6.claude-mem compresses session context
